# Email Notify Skill

Send one-way email notifications through a Gmail Apps Script webhook. Used for alerting on actionable infrastructure findings. This is outbound-only — not for two-way communication.

## Usage

```bash
python3 /data/skills/email-notify/scripts/email_notify.py --subject "[WARNING] Host issue" --body "Details here..."
```

## Options

| Flag | Required | Description |
|------|----------|-------------|
| `--subject` | Yes | Email subject line |
| `--body` | Yes | Email body text |
| `--to` | No | Recipient (default: krussell1@baldwinmutual.com) |

## Severity-to-Subject Mapping

Use these prefixes in the email subject line:

| Zabbix Severity | Subject Prefix | When to Use |
|-----------------|---------------|-------------|
| 5 (Disaster) | `[CRITICAL]` | Service down, data loss risk |
| 4 (High) | `[CRITICAL]` | Major degradation |
| 3 (Average) | `[WARNING]` | Notable issue, investigate within hours |
| 2 (Warning) | `[WARNING]` | Anomaly, may self-resolve |
| 1 (Information) | `[INFO]` | Informational only |

## Email Body Format

```
Finding: <one-sentence summary>

Affected host: <hostname> (<IP>)

What was checked:
- <list of systems queried>

Root cause assessment:
- <what you think is happening>
- <confidence: confirmed / likely / uncertain>

Recommended action:
- <specific steps>

This alert was generated by inframon at <timestamp>.
```

## Do NOT Email (Log Only)

- Information-only severity (sev 1) unless correlated with higher severity
- Known-down hosts (pve-r620 is powered off intentionally)
- Transient spikes that resolve within same check cycle
- Duplicate alerts already notified recently
- Hosts in Zabbix maintenance mode

## Environment Variables

- `NOTIFY_WEBHOOK_URL` — Apps Script deployment URL
- `NOTIFY_WEBHOOK_API_KEY` — API key for webhook auth
